Open the Soar Perspective

An Eclipse perspective is a pre-defined grouping of views associated with the language or problem context we want to work with. Eclipse comes with perspectives useful for various tasks, the default being the Java Perspective.

To view the list of available perspectives, we will choose Window -> Open Perspective -> Other.

This displays a dialog box asking us to choose a perspective to open. In this case, we will be writing a Soar program, so we will open the Soar Perspective.

Select Soar and click OK.

Create a New Soar Project

In Eclipse, files exist within projects. Before we create our Soar file, we must first create a new project for that file to be a part of.

To create a new project, right-click in the Package Explorer view and choose New -> Project.

This opens the New Project wizard dialog. This dialog contains wizards for every type of project Eclipse knows about. In order to provide support for Soar within multiple project types, there is no specific "Soar" project type.

Since we will just be writing Soar code, choose General -> Project from the list of project wizards and click Next.

Note: If we wanted to integrate Soar into a Java project, we would instead create a Java project here. This applies, in general, to other languages as well.

Now we need to name the new project. Type "helloworld" in theProject name box.

Once this is done, click Finish to create the project.

The newly created project is now displayed in the Package Explorer.

As we said, support for Soar can be provided in any type of project, but to get the project to provide support Soar, we must explicitly tell it to do so.

To do this, simply right click on the project in the Package Explorer and choose Include Soar Project Support.


Note: We can tell that a project has Soar Support by the "s" decoration on the project icon in the Package Explorer as shown below.

Create a New Soar File

Now that we have an empty project with Soar support, we can create a new Soar file for that project.

To add a new file to a project, right-click on the helloworld project and choose New -> File.

The New File dialog should now appear on the screen. This dialog handles the creation of most types of files, with the file type specified by the filename's extension.

Now we need to name the new file "hello.soar".

Once this is done, click Finish to create the file.

Note: The helloworld project is automatically selected as the parent folder when the file is created in this way.

Write a Soar Production

Once we have created our new empty Soar file, we can see that it is automatically opened in an editor.

To write our Soar production, we use the "sp" template built in to the IDE. Simply type "sp" into the editor and then press ctrl-space. This displays the auto-complete popup as shown to the right.

Open the "sp" template by double-clicking on it or by pressing Enter.

Opening the "sp" template generates a bare-bones Soar production. The production name is highlighted, ready to be entered.

Enter a production name, then press tab to jump to the next outlined box in the template.

Continue to fill out the rest of the production as shown.

IDE Overview

Run the Soar Project

To run the Soar project, we need to hook it to a debugger. In this case, we will use the Java Soar Debugger available as part of the Soar Suite.

To hook the project to the Java Soar Debugger, we will use Eclipse's External Tools run manager. This is accessed by clicking the arrow next to the External Tools icon in the Eclipse toolbar, then choosing External Tools from the dropdown menu.

The External Tools dialog should now appear on the screen.

To create a new launch configuration, we will click the New button at the far left of the toolbar.

Note: Make sure the Program launch configuration is highlighted, not the Ant Build launch configuration.

We need to give the new launch configuration four pieces of information to run.

  • Name: This can be anything, in this case "java_soar_debugger".
  • Location: To keep this line clean, we will be running a .bat (or .sh) script. Here is a sample one.
  • Working Directory: The working directory for the script (or program) to run from.
  • Arguments: The arguments to give the program that is being run.

Now save the run configuration by clicking Apply and run it by clicking Run. Running it opens the Java Soar Debugger with the production loaded and ready to go.

Note: Once we have run the configuration once, it will appear in the External Tools dropdown menu on the Eclipse toolbar.
